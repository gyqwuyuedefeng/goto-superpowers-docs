# 通用任务真实终态与 R 运行时依赖 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `hybrid-subagent-driven-development` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让 `goto/task` 通用任务的 Job Ledger、通知和 Redis Stream ACK 基于后台真实执行结果，并为 `gotoTools` 增加统一 R 运行时依赖预检。

**Architecture:** 当前 Codex 会话作为 controller，负责 worktree 准备、任务拆分、DeepSeek 任务派发、代码审查、测试确认和用户决策。DeepSeek 通过 `codex-dp-exec` 在隔离 worktree 中逐任务实现；`freedom-public` 提供共享线程池完成回调，`goto` 接入真实终态与 ACK，`gotoTools` 提供统一 R 依赖检查。

**Tech Stack:** Java 17、Spring Boot 3.3.4、Spring Data JPA、Redisson Redis Stream、`redis-stream-spring-boot-starter`、Maven、R package、Rserve、Docker、codex-dp-exec。

## 字段/接口/约束映射

### Job Ledger 终态顺序

成功终态：

```text
后台 CommonTaskRunnable 真实完成
→ CommonTaskRedisCompletionHandler 校验 ownerId + fencingToken
→ JobLeaseService.markSucceeded(jobId, ownerId, fencingToken, resultJson)
→ TaskEventPublisher.publishNotification(...)
→ AckHandle.ack()
```

最终失败终态：

```text
后台 CommonTaskRunnable 返回失败或抛出不可恢复异常
→ 错误分类为 FAILED_FINAL
→ JobLeaseService.markFinalFailure(jobId, ownerId, fencingToken, errorMessage)
→ TaskEventPublisher.publishNotification(...)
→ AckHandle.ack()
```

可重试失败：

```text
后台异常分类为 FAILED_RETRYABLE 或完成时失权
→ 能安全写入时 markRetryableFailure
→ 不发布最终成功通知
→ 默认不 ACK，等待租约恢复链路
```

### CommonTaskCompletionContext

建议字段：

```text
jobId: Long
ownerId: String
fencingToken: Long
messageId: String
consumerName: String
idempotencyKey: String
businessType: String
businessId: String
taskType: Integer
commonNoticeUUID: String
ackHandle: AckHandle
submittedAt: Instant 或 LocalDateTime
retryCount: Integer
```

稳定 registry key：

```text
consumerName + ":" + idempotencyKey
```

不得只用线程名作为 key。

### AckHandle

只暴露：

```text
void ack()
boolean isAcked()
String messageId()
```

实现可以包装 `MessageContext`，也可以保存 streamKey/group/messageId 后通过 Redisson ACK。第一版允许包装 `MessageContext`，但业务完成处理器不得直接持有完整 `MessageContext`。

### OwnerIdProvider

新增配置：

```yaml
task:
  instance:
    service-id: goto-task
```

ownerId 格式：

```text
{serviceId}:{nodeName}:{bootUuid}
```

默认 serviceId 为 `goto-task`，保持兼容。

### R 运行时依赖返回协议

`gotoTools` 统一依赖检查失败必须返回现有 Java 可解析的 `RInvokeResponse` JSON：

```json
{
  "status": "fail",
  "msg": "缺少运行时依赖: pandoc",
  "data": {
    "missing": [
      {"type": "tool", "name": "pandoc", "action": "install pandoc in Rserve container"}
    ]
  }
}
```

具体 `status` 字符串必须与现有 `RInvokeResult` / `returnStatus` 约定一致。实现前必须查当前 `gotoBase::returnStatus` 或等效函数，不能猜测。

```bash
git add final-review-handoff.md

```

## Fix Prompt Requirements

当 controller review 发现问题时，fix prompt 必须自包含，不能引用“上面”。包含：

```text
Original Task:
- task_id
- title
- goal
- worktree
- editable files
- forbidden changes
- required tests

Current State:
- git diff summary
- changed files
- latest commit hash if any
- previous report status
- tests already run

Review Findings To Fix:
- Spec compliance findings
- Code quality findings

Allowed Fix Scope:
- files and behavior the fix may touch
- explicit out-of-scope items

Required Tests:
- exact commands to rerun

Self Review Before Final Report:
- verify listed findings are fixed
- verify no unrelated changes were introduced
- verify tests were rerun
```

## Hybrid readiness checklist

- [ ] 每个批准的 spec 目标都映射到任务。
- [ ] 每个 DeepSeek 任务都有稳定 ASCII `task_id`。
- [ ] 每个 DeepSeek 任务都有明确 worktree、可编辑文件、禁改范围、测试命令、提交命令和报告要求。
- [ ] Task 0 记录 base branch、base commit、feature branch、worktree path 和 creation_status。
- [ ] 所有任务禁止 `mvn deploy` 和数据库写操作。
