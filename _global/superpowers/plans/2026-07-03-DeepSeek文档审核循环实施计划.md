## Hybrid Subagent-Driven 执行约定

执行前 controller 先检查：

```bash
/home/gyq/.local/bin/codex-dp-exec doctor --cd /mnt/g/Obsidian/skill/superpowers
```

DeepSeek 任务调用模板：

```bash
/home/gyq/.local/bin/codex-dp-exec \
  --cd /mnt/g/Obsidian/skill/superpowers \
  --prompt-file <prompt-file> \
  --task-id <task-id> \
  --controller-id <codex1-or-codex2-or-codex-dp-or-deepseek-main> \
  --output-last-message <report-file>
```

每个 DeepSeek run 后 controller 必须执行：

```bash
/home/gyq/.local/bin/codex-dp-exec status \
  --cd /mnt/g/Obsidian/skill/superpowers \
  --task-id <task-id>
```

Controller 检查顺序：

1. `codex-dp-exec status` 输出的 run 状态、report、stdout、stderr、command 文件。
2. DeepSeek 最终报告。
3. `git diff`、`git status --short` 和实际改动文件。
4. 测试命令输出。
5. 先做 spec compliance review。
6. spec compliance 通过后再做 code quality review。
7. 两阶段 review 都通过后才允许提交或进入下一任务。

恢复规则：

- 如果 run 为 `interrupted`，controller 先检查当前 `git diff`，再决定保留改动、重跑同一 `task_id` 的新 run，或向用户升级。
- 如果 DeepSeek 返回 `BLOCKED` 或 `NEEDS_CONTEXT`，controller 判断是补上下文、拆任务、重试，还是询问用户。
- 临时 prompt、report 和 run 目录不得提交，除非用户明确要求保留审核证据。
- 不得回退用户或其他 agent 的无关改动。

Important implementation details:

```bash
set -euo pipefail
sanitize_id() {
  local value="$1"
  value="${value//[^A-Za-z0-9._-]/-}"
  value="${value##-}"
  value="${value%%-}"
  [[ -n "$value" ]] || value="document"
  printf '%s\n' "$value"
}
```

Use `mktemp -d` for prompt generation and `trap 'rm -rf "$tmp_dir"' EXIT` so normal runs do not leave prompt files behind.
