# 开始分析前列名标记校验实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在开始分析前统一校验必选列名标记，前端即时拦截，后端接口兜底拒绝非法请求。

**Architecture:** 前端继续在 `draw.vue` 的提交流程前检查 `userDraw.fileSettings` 中必选 `SELECT_COLUMN` 标记是否已完成；后端在 `PlotServiceImpl.customDrawPlot(...)` 进入任务提交流程前复用同口径校验。测试分别覆盖前端拦截和后端失败返回，确保提示文案统一为“请先标记列名”。

**Tech Stack:** Vue 2、Jest、Java、JUnit 5、Mockito、Spring Boot

### Task 3: 实现后端列名标记校验

**Files:**
- Modify: `goto/system/src/main/java/com/freedom/service/impl/PlotServiceImpl.java`

- [ ] **Step 1: 在 `customDrawPlot(...)` 中加入校验入口**

实现要求：
- 放在参数异常校验之后、`UserDrawUtils.userDrawUpdate(componentBean, plotType)` 之前
- 校验失败时直接 `return ResponseUtils.failureMsg("请先标记列名")`

- [ ] **Step 2: 抽出最小私有方法，统一判定逻辑**

建议新增私有方法：
- `hasMissingRequiredColumnMark(List<ChartUploadFileSettings> fileSettings)`
- `hasRequiredColumnMark(ChartUploadFileSettings fileSetting, FileMarkContainer requiredMark)`

判定规则：
- 仅检查 `type=SELECT_COLUMN && required=true`
- `headerInfo.status` 在 `positionParam` 对应位为 `1` 视为已标记
- 没有 `fileMarkContainerList` 或没有必选项时不拦截

- [ ] **Step 3: 保持失败路径无副作用**

确保失败时：
- 不执行 `UserDrawUtils.userDrawUpdate(...)`
- 不设置 `paramSettingsList`
- 不发送 Redis Stream 消息
