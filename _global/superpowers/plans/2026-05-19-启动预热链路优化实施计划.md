# 启动预热链路优化实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 缩短 `goto/system` 启动阶段的数据库预热耗时，避免预热逻辑在启动时放大查询风暴和连接占用。

**Architecture:** 将启动预热拆成“只保留首页必须项的同步预热”和“后台补全详情的异步预热”。PlotGroup 关联缓存改为只提取文件名字段，不再把整份 `paramSettings` 递归转成 `Param`；字典选项改为本地按 `dictId` 复用，避免在预热阶段对同一字典重复查库。

**Tech Stack:** Spring Boot 3.3, JUnit 5, Mockito, JPA, Redisson, Caffeine, Jackson

---

private static String resolvePlotTypeFileName(List<ParamContainer> paramSettings, ComponentBean componentBean) {
    if (ObjectUtils.isEmpty(paramSettings)) {
        return null;
    }
    for (ParamContainer paramContainer : paramSettings) {
        if (paramContainer == null) {
            continue;
        }
        if (PlotTypeUtils.FILE_NAME_FIELD_NAME.equals(paramContainer.getTypeName())) {
            return extractValueFromContainer(paramContainer);
        }
    }
    return null;
}
