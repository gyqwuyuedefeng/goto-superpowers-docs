# Redis Stream 跨服务通知与长任务稳定性 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `hybrid-subagent-driven-development` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将 `goto` 与 `freedom-public` 的服务间异步通信统一到 Redis Stream，删除 RabbitMQ，并在长任务链路中补齐 Job Ledger、租约、fencing token 和通知版本过滤。

**Architecture:** 当前 Codex 会话作为 controller，负责执行顺序、任务拆分、代码审查、测试确认和用户决策；DeepSeek implementer 通过 `codex-dp-exec` 逐任务改代码。计划按三个独立 Git 仓库拆分 worktree：`freedom-public` 处理共享抽象，`goto` 处理后端任务和通知链路，`goto-web` 处理前端乱序过滤与兼容验证。

**Tech Stack:** Java 17、Spring Boot 3.3.4、Spring Data JPA、Redisson Redis Stream、`redis-stream-spring-boot-starter`、Vue 2、Element UI、Maven、npm、codex-dp-exec。

```text

## Task 1: freedom-public 移除 RabbitMQ 通知抽象

- [ ] task_id: `task-001-freedom-public-remove-rabbitmq-notice`
- [ ] Execution mode: `DeepSeek implementer + controller重点复核`
- [ ] `--cd`: `/mnt/f/IdeaProjects/goto-software/worktrees/freedom-public/feat/001-remove-rabbitmq-notice`

### Why This Task Exists

跨服务通知通道收敛设计要求 RabbitMQ 在共享模块中下线。如果 `freedom-public` 继续暴露 `NoticeType.RABBITMQ` 或 RabbitMQ 通知分支，`goto` 后续仍可能通过共享线程池回到旧通道。本任务先清理共享抽象，为 `goto` 切 Redis Stream 提供基础。

### Editable Files

```

### Forbidden Changes

- 不修改 `common-websocket` 的 endpoint 行为。
- 不删除 `NoticeType.WEBSOCKET` 和 `NoticeType.REDIS_STREAM`。
- 不引入 RabbitMQ 兼容开关。
- 不执行 `mvn deploy`。

### Existing Context

`NoticeType` 当前包含 `WEBSOCKET`、`RABBITMQ`、`REDIS_STREAM`。`NoticeRelationData` 暴露 `exchange`、`routingKey`、`streamKey`。`common-thread` 的 `ThreadPool` 只支持 WebSocket/RabbitMQ 通知分支；`common-thread-r` 的 `CommonStage` 已有 Redis Stream 分支，但仍保留 RabbitMQ 分支。

### Required Implementation Steps

1. 删除 `NoticeType.RABBITMQ` 枚举值。
2. 将 `NoticeRelationData` 改为通用通知配置，保留 `streamKey`，移除或停止使用 `exchange`、`routingKey`。
3. 将 `ThreadPool` 的提交成功、提交失败、取消通知分支改为支持 `WEBSOCKET` 和 `REDIS_STREAM`，不再调用 RabbitMQ。
4. 删除 `CommonStage` 中 RabbitMQ 通知分支，保留并整理 Redis Stream 通知路径。
5. 如需要抽象 Redis Stream 发布，使用 `redis-stream-spring-boot-starter` 现有 `StreamProducerFactory` / `StreamTemplate`，不要新增业务专属依赖到共享模块。
6. 删除共享模块中不再需要的 AMQP import 和依赖。
7. 修改或新增相关单元测试；所有新增/修改 Java 测试类和测试方法必须有详细中文注释。
