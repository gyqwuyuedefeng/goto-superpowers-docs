# goto 过期路径配置硬清理实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将 `goto` 项目中与多拓扑路径重构相关、且已在 `RProperties` 中标记 `@Deprecated` 的旧路径配置完成一次硬清理，并保留仓库内备份。

**Architecture:** 先备份即将修改的公共配置文件，再通过定向测试锁定旧字段清理后的行为边界，随后删除 `RProperties` 中对应字段与公共 YAML 中对应配置项，并把所有生产代码、测试代码和当前文档统一收敛到 `PathResolver + LogicalPath + PathTarget` 新模型或仍保留的非废弃字段上。

**Tech Stack:** Java 17, Spring Boot, Maven, JUnit 5, Mockito, GitNexus, YAML 配置

## Sequencing Rules

- 所有修改过的 Java 测试文件都必须补齐详细中文类注释和方法注释。
- 每个任务必须先写或调整失败测试，再执行最小实现，再跑定向验证。
- 先做检索和备份，再删字段，避免一次删掉后丢失对照。
- 删除字段时必须通过全仓库检索和编译双重校验，不能凭肉眼判断“应该没人用了”。
- 如执行中发现某个字段虽然标记废弃，但仍承担未迁移的生产责任，应立即停止实施并回到设计阶段，不允许静默保留半删状态。
