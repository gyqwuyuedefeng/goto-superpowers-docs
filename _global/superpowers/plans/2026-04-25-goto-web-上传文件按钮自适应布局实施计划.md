# goto-web 上传文件按钮自适应布局 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 仅在 `goto-web` 绘图侧边栏的“上传文件”区域实现自适应布局，使上传按钮在宽容器下自动并列、窄容器下自动切换为单列 `100%`，且不影响其他参数模块样式。

**Architecture:** 实现集中在 [PlotSidePanel.vue](/mnt/f/IdeaProjects/goto-software/goto-web/src/views/goto/plot/type/components/PlotSidePanel.vue) 的上传文件折叠区，通过新增上传专用 class 把布局职责收敛到局部容器。按钮组件 [uploaderBtn.vue](/mnt/f/IdeaProjects/goto-software/goto-web/src/views/goto/components/form/draw/uploaderBtn.vue) 继续只负责按钮内部内容排布；容器使用 CSS Grid 的 `auto-fit/minmax` 控制并列与单列切换，避免修改公共表单栅格。

**Tech Stack:** Vue 2 SFC, SCSS, Element UI, Git, GitNexus CLI（仅作可用范围核对）, 手工页面回归

## Task 2: 用自适应 Grid 替换上传区域的强制纵向堆叠

**Files:**
- Modify: `goto-web/src/views/goto/plot/type/components/PlotSidePanel.vue`

- [ ] **Step 1: 先写出目标样式骨架，保留内部 100% 宽度链路**

Use these code shapes:

```scss
.plot-upload-grid {
  display: grid !important;
  grid-template-columns: repeat(auto-fit, minmax(240px, 1fr));
  gap: 12px;
  width: 100% !important;
  min-width: 0;
  padding: 2px;
}

.plot-upload-item {
  display: flex !important;
  width: 100% !important;
  min-width: 0;
}
```

Expected: 上传区域容器从固定纵向 `flex-direction: column` 切到自适应 grid；每个上传项仍保持 `width: 100%`。

- [ ] **Step 2: 清理或下沉只会阻止并列的旧规则，但不要删除上传区需要的内部撑满规则**

Keep these kinds of rules under `.plot-upload-item`:

```scss
.plot-upload-item {
  ::v-deep .container,
  ::v-deep .el-upload-container,
  ::v-deep .uploader,
  ::v-deep .el-upload,
  ::v-deep .el-upload-dragger {
    width: 100% !important;
    min-width: 0;
  }
}
```

Expected: 仅移除“强制纵向堆叠”的限制，不破坏 `Uploader` 与 `GUploaderBtn` 的满宽行为。

- [ ] **Step 3: 为最小可读宽度做一次收敛，必要时微调 `minmax` 下限**

Decision rule:

```text
如果 240px 下文件名、图标、删除按钮仍不挤压，则保持 240px；
如果已上传态显著拥挤，则调到 260px 或 280px，但不要引入固定媒体查询。
```

Expected: 通过 `minmax` 下限控制单元最小可用宽度，而不是写死 `@media` 断点。

- [ ] **Step 4: 查看局部 diff，确认样式只挂在上传专用 class 上**

Run:

```bash

```

Expected: 新增的 grid 规则只绑定 `plot-upload-grid` / `plot-upload-item`，没有重定义公共参数布局类的语义。

- [ ] **Step 5: 提交上传区域自适应布局实现**
