# goto-web 韦恩图增量保存误报 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 修复韦恩图在增量保存真实成功后仍误弹“增量保存失败，是否尝试全量保存？”的问题，并确保保存后联动异常不再被误判为接口失败。

**Architecture:** 前端改动限制在 [vennModify.js](/mnt/f/IdeaProjects/goto-software/goto-web/src/constant/modify/vennModify.js) 与 [handsontable.vue](/mnt/f/IdeaProjects/goto-software/goto-web/src/views/components/handsontable.vue) 两处。第一处通过判空保护消除韦恩图可选分组缺失导致的回调异常；第二处通过将 `saveFileSuccess()` 从接口成功链路中隔离出来，纠正“保存成功”和“保存后 UI 联动失败”的错误归因。

**Tech Stack:** Vue 2 SFC, Jest + Vue CLI Unit Test, Element UI, GitNexus MCP, Git

```js
describe('vennSaveFileSuccess', () => {

        visible: false,
        paramSettingsList: []
      })
    }

    if (includeUpsetGroup) {
      topParamSettingsList.push({
        moduleType: -1,
        name: '视觉属性（Upset韦恩图）',
        visible: false,
        paramSettingsList: []
      })

    return {
      fileSettingIndex: 0,
      fileSettings: [
        {
          headerInfoList: [
            { columnName: 'set_a', status: 1, choose: true },
            { columnName: 'set_b', status: 1, choose: true },
            { columnName: 'set_c', status: 1, choose: true }
          ]
        }
      ],
      topParamSettingsList,
      vueContext: {
        $set(target, key, value) {
          target[key] = value
        }
      }
    }
  }

  test('does not throw when flower and upset groups are both missing', () => {
    const commonParams = createCommonParams({
      includeFlowerGroup: false,
      includeUpsetGroup: false
    })

    expect(() => vennSaveFileSuccess(commonParams)).not.toThrow()
  })

  test('updates visible flags when optional groups exist', () => {
    const commonParams = createCommonParams()

    vennSaveFileSuccess(commonParams)

    const flowerGroup = commonParams.topParamSettingsList.find(
      item => item.name === '视觉属性（花瓣图）'
    )
    const upsetGroup = commonParams.topParamSettingsList.find(
      item => item.name === '视觉属性（Upset韦恩图）'
    )

    expect(flowerGroup.visible).toBe(false)
    expect(upsetGroup.visible).toBe(true)
  })
})
```

```js
import crudPlotType from '@/api/plotType'
import Handsontable from '@/views/components/handsontable.vue'

describe('handsontable saveIncremental', () => {
  function createVm() {
    return {
      plotType: { groupId: 6, id: 1001 },
      userDrawId: 2001,
      tmpFolderName: 'tmp-folder',
      example: false,
      fileSettingIndex: 0,
      fileSettingsName: 'fileSettings',

      },

        ['0-columnA', { row: 0, column: 'columnA', newValue: 'new', oldValue: 'old' }]
      ]),
      changeHistory: [],
      lastSaveVersion: 0,
      saveLoading: false,
      ToastificationNotice: {
        notifySuccess: jest.fn(),
        notifyError: jest.fn(),
        notifyWarning: jest.fn()
      },
      $confirm: jest.fn(() => Promise.resolve()),
      $refs: {
        hotTable: {
          hotInstance: {
            getSourceData: jest.fn(() => [{ columnA: 'new' }])
          }
        }
      },
      obtainCommonParams: jest.fn(() => ({})),
      updateModifiedCellStyles: jest.fn(),
      saveFileSuccess: jest.fn(() => {
        throw new Error('post-save modify failed')
      }),
      saveFile: jest.fn()
    }
  }

  beforeEach(() => {
    jest.clearAllMocks()
  })

  test('keeps success result when post-save callback throws', async () => {
    crudPlotType.saveFileSettingIncrementalHandsontable.mockResolvedValue({
      code: 200,
      msg: '保存成功'
    })

    const vm = createVm()

    await Handsontable.methods.saveIncremental.call(vm)
    await Promise.resolve()

    expect(vm.ToastificationNotice.notifySuccess).toHaveBeenCalledWith(vm, '保存成功')
    expect(vm.$confirm).not.toHaveBeenCalled()
    expect(vm.saveFile).not.toHaveBeenCalled()
    expect(vm.ToastificationNotice.notifyError).not.toHaveBeenCalledWith(
      vm,
      expect.stringContaining('增量保存异常')
    )
  })

  test('still offers full-save fallback when request rejects', async () => {
    crudPlotType.saveFileSettingIncrementalHandsontable.mockRejectedValue({
      response: { status: 500 }
    })

    const vm = createVm()
    vm.saveFileSuccess = jest.fn()

    await Handsontable.methods.saveIncremental.call(vm)
    await Promise.resolve()

    expect(vm.$confirm).toHaveBeenCalled()
  })
})
```

- [ ] **Step 3: 在 `handsontable.vue` 中实现最小异常隔离**

Refactor the success branch in `goto-web/src/views/components/handsontable.vue`:

```js
if (res.code === 200) {
  this.ToastificationNotice.notifySuccess(this, '保存成功')
  this.clearChangeHistory()
  this.updateLocalDataAfterSave()

  try {
    this.saveFileSuccess()
  } catch (postSaveError) {
    console.error('保存成功后的联动处理异常:', {
      error: postSaveError,
      groupId: this.plotType && this.plotType.groupId,
      plotTypeId: this.plotType && this.plotType.id,
      fileSettingIndex: this.fileSettingIndex
    })
  }
} else {
  const errorMsg = res.msg || res.message || '增量保存失败！'
  this.ToastificationNotice.notifyError(this, errorMsg)
}
```

Implementation rules:

1. `saveFileSuccess()` 的异常不能再进入当前外层 `.catch()`
2. 真正的接口拒绝或 HTTP 异常仍然保留现有全量保存兜底
3. 非 `200` 分支优先取 `res.msg`，兼容当前后端 `ResponseUtils.failureMsg(...)` 返回结构
