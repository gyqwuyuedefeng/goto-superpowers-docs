# 多智能体文档审核系统 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `hybrid-subagent-driven-development` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将现有的单一 DeepSeek 审核脚本改造成多 reviewer 并行审核 + AI aggregator 汇总的系统，替换 `deepseek-review-spec` 和 `deepseek-review-plan` 中的单次调用模式。

**Architecture:** 当前 Codex 会话是 controller，负责计划审查、任务拆分、审查和用户决策。DeepSeek 通过 `codex-dp-exec` 在 `/mnt/g/Obsidian` 仓库中执行脚本修改和 prompt 文件部署。每个 DeepSeek 任务自包含上下文。脚本改造后，每个审核阶段并行派发 6（spec）/7（plan）个 reviewer，各自独立审核，再由 aggregator 汇总成一份结构化报告。

**Tech Stack:** Bash、Git/worktree、`codex-dp-exec`、DeepSeek 审核者、已批准的 v2 reviewer prompts。

- 临时 prompt、report、run 目录不得提交，除非用户明确要求。

### 并行执行细节

```bash
# 每个 reviewer 的 codex-dp-exec 调用模式
for reviewer in spec-requirement-alignment ... spec-yagni-scope-control; do
  temp_prompt=$(mktemp)
  cat "$reviewers_dir/shared/common-reviewer-rules.md" > "$temp_prompt"
  cat "$reviewers_dir/spec/$reviewer.md" >> "$temp_prompt"
  printf '\n\n# Document To Review\n\n' >> "$temp_prompt"
  cat "$spec" >> "$temp_prompt"
  # 附加上下文（如有）
  codex-dp-exec --cd "$project_root" --prompt-file "$temp_prompt" \
    --task-id "review-$sanitized_id-$reviewer" \
    --controller-id "$controller_id" \
    --output-last-message "$report_dir/$reviewer-report.md" &
done
wait
```

### 失败处理

- 任何 reviewer 返回非零退出码 → aggregator 报告中标记该 reviewer 为 "Malformed or Missing"。
- 超时保护：每个 reviewer 调用可指定最大等待时间（如 300s）。
- 部分 reviewer 失败不应阻塞其他 reviewer 和 aggregator。
