# goto 多拓扑路径与配置重构实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将 `goto` 项目的路径与运行环境配置从“多份重复 YAML + 业务代码手工拼路径”重构为“稳定拓扑声明 + 统一路径解析层 + 渐进迁移”。

**Architecture:** 保留 `application-home.yml`、`application-public.yml`、`application-wsl.yml` 三个现有 profile 文件名，但将其职责收缩为声明拓扑与少量默认值。新增 `RuntimeTopology`、`PathTarget`、`LogicalPath`、`PathResolver` 等抽象，由公共模块集中处理 Java 本地路径与 R 远端路径的差异，再以 `DataProcess` 链路为第一批迁移切片，逐步替换 `separator`、`xxxForR` 与派生路径配置。

**Tech Stack:** Java 8, Spring Boot, Maven 3, Lombok, JUnit, existing `goto-common` / `goto-system` / `goto-task` modules

---

- 事实源设计文档是 `docs/superpowers/specs/2026-04-20-goto-多拓扑路径与配置重构设计.md`，本计划不得重新打开已确认的架构方向。
- 外部 profile 文件名必须保持不变：
  - `goto/task/src/main/resources/application-home.yml`

  - `goto/system/src/main/resources/config/application-wsl.yml`
  - `goto/system/src/main/resources/config/application-product.yml`
- 公共配置源仍在：
  - `goto/common/src/main/resources/application-common-home.yml`
  - `goto/common/src/main/resources/application-common-public.yml`
  - `goto/common/src/main/resources/application-common-wsl.yml`
- 本轮改造优先收口“跨 Java/R 边界的路径问题”，不是一次性重写所有 `File.separator` 使用点。
- 需要优先覆盖的真实拓扑只有 3 个：
  - Windows Java -> WSL Docker R
  - WSL Java -> WSL Docker R
  - Linux Docker Java -> Linux Docker R
- 计划中的 Maven 命令禁止使用 `deploy`；统一使用 Maven 3 和本项目 JDK 1.8。

## Sequencing Rules

- 所有新测试类与测试方法必须补齐详细中文注释。

- 每个任务必须先写失败测试，再做最小实现，再跑定向验证。
- 第一批只迁移 `DataProcess` 相关路径，验证模式稳定后再扩展到更多逻辑路径。
- 在 `PathResolver` 稳定前，允许 `RProperties` 与新配置模型短期并存，但禁止继续新增新的 `xxxForR` 字段。
- 若迁移中发现某些历史工具流依赖旧字段，先加兼容映射，不要立即删除旧绑定。
