# WeChat History List PNG Request Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the mini-program history-list request always send `imageType: "png"` to `POST /api/userDraw/page_list`, so chart detail pages receive PNG base64 when backend PNG results exist.

**Architecture:** Keep the fix entirely in the mini-program request layer. The backend `imageType` support already exists, so the only production behavior change is to have `chartApi.getHistoryList` merge a fixed `imageType: "png"` into the request body. Validate this with a focused API-layer test and a light regression run for the history page tests.

**Tech Stack:** WeChat Mini Program, CommonJS, Node.js built-in test runner, existing `mock-wx` helpers

```js
test("chartApi.getHistoryList posts page_list with fixed png imageType", async () => {

  let capturedData = null;

  installMockWx({
    requestImpl({ url, data, success }) {
      capturedUrl = url;
      capturedData = data;
      success({ statusCode: 200, data: { code: 200, data: {} }, header: {} });
    },
  });

  const { chartApi } = freshRequireFromProject("utils/api");

  await chartApi.getHistoryList({
    page: {
      page: 2,
      pageSize: 10,
      sort: ["id,desc"],
    },
  });

  assert.equal(capturedUrl, "https://api.goto.com/api/userDraw/page_list");
  assert.deepEqual(capturedData, {
    page: {
      page: 2,
      pageSize: 10,
      sort: ["id,desc"],
    },
    imageType: "png",
  });
});
```
