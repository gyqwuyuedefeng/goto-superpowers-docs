# goto-web 图例设置联动设计

## 背景

当前图例相关配置分散在后端 `LegendSettingsParam` 模型、前端绘图页图例设置组件、配置页图例设置组件、分类图例刷新逻辑和分类图例绘制组件中。现有 `legendList` 条目只同步分类类型 `type`，无法区分连续颜色图例与连续大小、粗细图例，导致前端只能按“连续变量显示刻度数、离散变量显示列数”的粗粒度规则渲染控件。

本次需求需要完成三类调整：

1. 在全局图例设置里新增“对齐方式”参数。
2. 当分类图例数量为 0 时隐藏固定名称为“图例设置”的 tab。
3. 分类图例控件根据分类类型和数据类型决定显示“列数”或“刻度数”。

## 目标

- 后端 `LegendSettingsParam` 新增 `legendBoxJust` 字段，输出 R 参数名为 `legend.box.just`。
- `legendBoxJust` 使用字典 `71L`，默认值为 `left`，显示含义为“左对齐”。
- `draw/legendSettingsParam.vue` 与 `plotSetting/legendSettingsParam.vue` 都在“排列方式”后增加“对齐方式”控件。
- `classifyLegendRefresh` 刷新完 `legendList` 后，根据图例数量控制 `name === "图例设置"` 的分组显隐。
- `legendList` 条目补齐 `valueType` 信息，使 `classifyLegendParam.vue` 能区分连续颜色和连续非颜色。
- 连续颜色显示“刻度数”，连续大小、粗细显示“列数”，离散变量显示“列数”。

## 非目标

- 不新增或修改字典数据，只消费已有的 `71L` 字典。
- 不对找不到 `name === "图例设置"` 的情况做降级处理。
- 不按 `paramName === "legendSettings"` 隐藏单个参数。
- 不调整图例 R 参数生成框架的整体机制。
- 不批量重构现有图例组件布局。

## 后端模型设计

在 `goto/common/src/main/java/com/freedom/model/form/LegendSettingsParam.java` 中新增字段：

```java
@Syncable(nested = true)
@AutoParamField(name = "legend.box.just")
private InternalParameterItem<SelectParam> legendBoxJust;
```

构造器中按现有 `legendDirection` 模式初始化：

- 创建 `new InternalParameterItem<>(new SelectParam(), false, true)`。
- 通过 `CacheService.getGotoDictById(71L)` 获取“多个图例对齐方式”字典。
- 字典存在时写入 `dictId`、`dictName`、`selectList`。
- 默认 `value` 固定为 `left`。

字段使用 `@Syncable(nested = true)`，保持与配置同步、参数模板生成、前端配置读取的既有机制一致。

## 前端图例设置控件设计

### 绘图页组件

文件：`goto-web/src/views/goto/components/form/draw/legendSettingsParam.vue`

在 `legendDirection` 后新增 `legendBoxJust` 配置块：

- 外层使用 `SettingForm`。
- 内层使用 `GSelectObject`。
- 表单标签为“对齐方式”。
- 数据源使用 `setting.legendBoxJust.value.selectList`。
- 修改参数使用 `{ optionName: "legendBoxJust" }`。
- 事件沿用现有 `@modify="modify"`。

该组件只负责展示和修改默认值，不提供字典切换。

### 配置页组件

文件：`goto-web/src/views/goto/components/form/plotSetting/legendSettingsParam.vue`

在 `legendDirection` 后新增一行配置，结构与排列方式一致：

- 左侧为字典选择，标签“对齐方式”，字段 `dictId`。
- 右侧为默认值选择，标签“默认对齐方式”，字段 `value`。
- 同步字段命名为 `legendBoxJust_dictId` 与 `legendBoxJust_value`。
- 字典列表使用 `gotoDictList`。
- 默认值列表使用 `setting.legendBoxJust.value.selectList`。
- 修改事件沿用 `dictChange` 和 `selectDictChange`。

这样配置页可以切换字典和默认值，绘图页只消费最终配置。

## 图例设置 tab 显隐设计

文件：`goto-web/src/constant/classifyLegendRefresh.js`

显隐逻辑放在 `classifyLegendRefresh(commonParams)` 内部，理由是上传、删除、保存、参数联动等入口最终都会调用该刷新函数，集中处理可以避免遗漏调用点。

流程：

1. 查找 `legendList` 参数。
2. 收集分类参数并同步 `classifyLegendSetting.value`。
3. 计算图例数量：`classifyLegendSetting.value.length`。
4. 使用 `localFindGroupFromTopParamSettingsListByName("图例设置", commonParams)` 查找固定 tab 分组。
5. 找到分组时设置 `group.visible = classifyLegendSetting.value.length > 0`。
6. 找不到分组时不做任何降级处理。

该逻辑只控制固定名称分组，不隐藏 `legendSettings` 参数本身。

## 分类图例数据设计

当前 `classifyLegendRefresh` 使用 `by -> Set<type>` 聚合分类图例。新设计改为聚合更多信息，至少包含：

- `typeSet`：保留现有离散、连续聚合能力。
- `valueTypeSet`：记录同一 `by` 下涉及的数据类型。
- `continuousValueTypeSet`：记录连续分类对应的数据类型，用于判断是否存在连续颜色。

同步 `legendList` 时：

- 继续写入 `classifyLegendItem.type = resolveLegendType(typeSet)`。
- 新增写入 `classifyLegendItem.valueType`。
- 如同一 `by` 有多个 `valueType`，优先写入颜色 `1`，因为连续颜色图例需要刻度数。
- 同步写入数组型辅助字段 `valueTypeList` 与 `continuousValueTypeList`，用于前端准确判断聚合类型。

推荐前端判断优先使用辅助列表；若历史数据没有列表字段，则回退使用 `valueType`，保证旧数据不会直接失效。

## 分类图例控件显示设计

文件：`goto-web/src/views/goto/components/form/draw/classifyLegendParam.vue`

新增方法封装模板判断：

- `shouldShowNcol(classifyLegendItem)`
- `shouldShowNticks(classifyLegendItem)`
- `isContinuousColorLegend(classifyLegendItem)`

显示规则：

- 离散变量：显示“列数”。
- 连续变量且数据类型为颜色 `valueType === 1`：显示“刻度数”。
- 连续变量且数据类型不是颜色：显示“列数”。
- 聚合类型 `type === 3`：
  - 如果包含连续颜色，显示“刻度数”。
  - 因为同时包含离散图例，保留“列数”。
  - 如果不包含连续颜色，只显示“列数”。

模板不直接写复杂表达式，避免后续维护者难以判断离散、连续、聚合三类场景。

## 错误与兼容处理

- 字典 `71L` 未返回时，后端仍创建 `legendBoxJust` 参数对象，但不会填充字典列表；前端组件通过现有空对象判断避免渲染异常。
- 旧数据中没有 `legendBoxJust` 时，前端两个组件都按现有 `CommonUtils.isNotEmptyObject` 或 `SettingForm` 容器规则保护渲染。
- 旧 `legendList` 条目没有 `valueTypeList` 时，前端回退到 `valueType` 判断。
- 若没有找到 “图例设置” 分组，不抛错、不做降级隐藏。

## 测试设计

- 后端单元或编译校验：确认 `LegendSettingsParam` 能构造并包含 `legendBoxJust` 字段，默认值为 `left`。
- 前端静态检查：确认两个 `legendSettingsParam.vue` 组件能通过 lint 或构建解析。
- 刷新逻辑测试：
  - 没有任何有效分类图例时，“图例设置”分组 `visible=false`。
  - 有至少一个有效分类图例时，“图例设置”分组 `visible=true`。
  - 找不到 “图例设置”分组时函数正常结束。
- 分类图例展示测试：
  - 离散颜色显示列数。
  - 连续颜色显示刻度数。
  - 连续大小显示列数。
  - 连续粗细显示列数。
  - 同一 `by` 同时包含离散和连续颜色时同时显示列数与刻度数。

## 验收标准

- 后端返回的 `legendSettings` 中包含 `legendBoxJust`，R 参数名为 `legend.box.just`。
- 绘图页和配置页都能看到“对齐方式”控件，并默认选中 `left`。
- 当 `legendList.value.length === 0` 时，固定分组 “图例设置” 不显示。
- 当 `legendList.value.length > 0` 时，固定分组 “图例设置” 显示。
- 连续颜色分类图例显示刻度数，连续非颜色分类图例显示列数，离散分类图例显示列数。
