# 管理员通用统计页设计

## 背景

后端已完成“用户绘图类型每日统计”能力，并提供管理员查询热门图类型排行的接口：

- 接口：`POST /api/userPlotTypeStat/popular_plot_types`
- 权限点：`userPlotTypeStat:popular_plot_types`
- 返回字段：`plotTypeId`、`plotTypeName`、`groupId`、`groupName`、`drawCount`

当前前端 GOTO 顶部菜单只有“画图”和“工具”，还没有承载管理员数据查看的通用页面。本设计新增一个管理员可见的“管理”页面，首版展示用户画图情况，并为后续管理员统计模块保留清晰扩展位置。

## 目标

- 在 GOTO 顶部菜单“画图”右侧新增“管理”入口。
- “管理”入口仅在用户角色包含 `admin` 或 `userPlotTypeStat:popular_plot_types` 时显示。
- 新增独立管理页，首版展示用户画图热门图类型排行和汇总指标。
- 首版管理页隐藏左侧栏，让统计内容占满主区域。
- 只消费现有后端统计接口，不新增后端接口。

## 非目标

- 不新增用户排行、每日趋势、任务耗时、失败率等后端统计能力。
- 不复用当前已注释的 `/goto/statistics` 模块作为入口。
- 不引入新的 UI 组件库或独立主题体系。
- 不调整后端权限注解，接口访问控制仍由后端 `@PreAuthorize` 兜底。

## 路由与导航

新增 GOTO 子路由：

- 路径：`/goto/manage`
- 菜单文案：`管理`
- 顶部菜单激活值：`activeIndex = "5"`

`goto-web/src/views/goto/index.vue` 负责三项布局行为：

- 在“画图”右侧、“工具”左侧渲染“管理”顶部项。
- 点击“管理”时跳转 `/goto/manage`，并设置 `activeIndex` 为 `"5"`。
- 当当前路由为 `/goto/manage` 时隐藏左侧栏。

入口可见性判断使用 Vuex `roles`：

- `roles.includes("admin")`
- 或 `roles.includes("userPlotTypeStat:popular_plot_types")`

满足任一条件时显示“管理”入口。前端隐藏只是体验控制，不替代后端权限校验。

## 页面结构

新增页面组件建议路径：

- `goto-web/src/views/goto/manage/index.vue`

页面标题为“管理”，副标题为“管理员数据总览”。首版包含三块内容：

1. 时间范围筛选区
   - 使用 Element UI 单选按钮或分段按钮。
   - 选项为“最近 7 天”“最近 30 天”“最近 365 天”“全部”。
   - 对应请求值为 `LAST_7_DAYS`、`LAST_30_DAYS`、`LAST_365_DAYS`、`ALL`。
   - 默认值为 `LAST_7_DAYS`。

2. 汇总指标区
   - `总绘图次数`：对接口结果的 `drawCount` 求和。
   - `涉及图类型数`：接口结果数组长度。
   - `最高频图类型`：接口结果第一项的 `plotTypeName`，无数据时显示“暂无数据”。

3. 热门图类型区
   - 图表：使用项目已有 ECharts 依赖展示柱状图。
   - 表格：使用 Element UI 表格展示排行明细。
   - 表格列包含“排名”“图类型”“图分组”“绘图次数”。

页面代码结构上保留后续扩展空间，但首版不展示假数据，也不写无功能说明文案。

## API 封装

新增前端 API 文件：

- `goto-web/src/api/userPlotTypeStat.js`

导出方法：

```js
export function popularPlotTypes(data) {
  return request({
    url: "api/userPlotTypeStat/popular_plot_types",
    method: "post",
    data
  })
}
```

请求参数：

```json
{
  "rangeType": "LAST_7_DAYS",
  "limit": 20
}
```

首版默认 `limit` 使用 20。后续如果需要管理页配置排行数量，可以在页面筛选区增加数量选择。

## 数据流

管理页维护以下核心状态：

- `rangeType`：当前时间范围。
- `loading`：接口加载状态。
- `rankList`：热门图类型排行数组。

页面初始化时请求一次接口。用户切换时间范围时重新请求接口。

接口响应归一化规则：

- 如果返回值是数组，则直接作为排行数组。
- 如果返回值存在 `data` 且 `data` 是数组，则使用 `data`。
- 其他情况按空数组处理。

图表数据、表格数据和汇总指标都从 `rankList` 派生，避免重复状态。

## 异常与空状态

- 请求失败时停止 loading，清空 `rankList`，并使用 Element UI 提示“获取管理员统计数据失败”。
- 无数据时，汇总指标显示 `0` 或“暂无数据”。
- 图表区域显示空状态，不渲染误导性的假柱状图。
- 表格使用 Element UI 空状态展示。

## 视觉与交互原则

- 延续现有 GOTO 后台风格：白色内容块、紧凑信息密度、Element UI 控件。
- 首版隐藏左侧栏，主内容区全宽展示。
- 页面不做营销式大卡片，不引入装饰性背景。
- 字体、间距和边框应与现有 GOTO 页面保持一致。
- 时间范围切换应即时刷新数据，并在刷新期间展示 loading 状态。

## 测试与验证

前端验证：

- 对新增和修改的 Vue/JS 文件运行 lint，优先执行 `npm run lint -- --quiet`。
- 如果环境允许，执行 `npm run build:prod` 验证生产构建。
- 手动验证普通用户不显示“管理”入口。
- 手动验证 `admin` 用户显示“管理”入口并能进入页面。
- 手动验证只具备 `userPlotTypeStat:popular_plot_types` 权限的用户也显示入口。
- 手动验证接口失败、空数据、有数据三种状态。

后端验证：

- 本次设计不要求新增或修改后端代码。
- 如果实施中修改 Java 测试文件，必须按全局规则补充详细中文类注释和方法注释。

## 风险与约束

- 当前权限数据来自 Vuex `roles`，需要确认登录后 `roles` 中会包含细粒度权限点。若实际只包含角色名，入口将只能由 `admin` 控制，实施时需要根据现有权限模型调整判断来源。
- 现有接口只返回热门排行，不返回全量趋势数据。因此首版汇总指标只能基于排行结果计算，不代表系统全量绝对统计，除非 `limit` 足够覆盖全部图类型。
- 如果后续管理页模块增多，再考虑新增左侧管理子菜单；首版不提前引入。
