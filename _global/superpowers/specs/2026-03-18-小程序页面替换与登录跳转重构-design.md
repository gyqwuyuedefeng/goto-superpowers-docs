# 小程序页面替换与登录跳转重构设计文档

**日期：** 2026-03-18
**项目：** goto-wechat 小程序
**目标：** 用 pages/home 的内容完全替换 pages/index，并更新登录跳转逻辑

---

## 概述

本设计旨在重构小程序的页面结构，将功能列表页面（pages/home）替换当前的欢迎页面（pages/index），并确保登录成功后自动跳转到新的首页。

## 当前状态

### 页面结构
- **pages/home/index** - 功能列表页面
  - 显示：数据处理、统计分析、数据画图、实用工具等功能卡片
  - 当前是 app.json 中的首页（pages 数组第一项）

- **pages/index/index** - 欢迎页面
  - 显示：GOTO Software logo 和三个特性卡片
  - 存在但不在 app.json 的 pages 数组中

### 登录流程
- 登录成功后跳转到 `/pages/home/index`（login.js:180）

## 设计方案

### 方案选择：Git 重命名方式

**优势：**
- 保留文件的 Git 历史记录
- 操作简洁，步骤少
- Git 会自动追踪文件移动
- 可以轻松回滚

---

## 实施步骤

### 步骤 1：文件操作

**删除旧的 index 目录：**
```bash
rm -rf pages/index
```
- 清除旧的欢迎页面（GOTO Software logo 和三个特性卡片）

**重命名 home 目录为 index：**
```bash
git mv pages/home pages/index
```
- Git 会追踪这个移动操作，保留文件历史
- 结果：`pages/index` 现在包含原来的功能列表页面（数据处理、统计分析、数据画图、实用工具）

### 步骤 2：配置文件修改

**文件：`app.json`**

修改 pages 数组的第一项：
```json
"pages": [
  "pages/index/index",  // 从 "pages/home/index" 改为 "pages/index/index"
  "pages/assistant/index",
  "pages/workbench/index",
  "pages/my/index",
  "pages/common/detail",
  "pages/login/login"
]
```

**作用：**
- 将小程序的首页设置为新的 `pages/index/index`
- 小程序启动时会自动加载 pages 数组的第一项作为首页

### 步骤 3：登录跳转逻辑修改

**文件：`pages/login/login.js`（第 179 行）**

修改跳转路径：
```javascript
// 修改前
wx.redirectTo({
  url: "/pages/home/index",
  // ...
});

// 修改后
wx.redirectTo({
  url: "/pages/index/index",
  // ...
});
```

**作用：**
- 登录成功后自动跳转到新的首页
- 保持原有的成功/失败回调逻辑不变

### 步骤 4：全局引用检查

**搜索策略：**
在整个 `goto-wechat` 项目中搜索字符串 `pages/home`，检查以下可能的引用位置：
- 导航跳转代码（`wx.navigateTo`, `wx.redirectTo`, `wx.switchTab` 等）
- 配置文件（`*.json`）
- 注释中的路径引用
- 其他页面的跳转逻辑

**处理方式：**
- 将所有找到的 `pages/home/index` 替换为 `pages/index/index`
- 如果发现注释中的引用，也一并更新以保持文档准确性

## 回滚方案

如果出现问题，可以使用 Git 回滚：

**方式 1：回滚到上一次提交**
```bash
git reset --hard HEAD~1
```

**方式 2：使用 reflog 找到操作前的状态**
```bash
git reflog  # 查看操作历史
git reset --hard <commit-hash>  # 回滚到指定提交
```

## 风险评估

**风险等级：** 低

- 文件操作使用 `git mv`，保留完整历史
