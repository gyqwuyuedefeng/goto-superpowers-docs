# 小程序数据画图功能设计文档

**日期：** 2026-03-19
**目标：** 为小程序添加数据画图历史记录查看功能，并标记开发中的功能模块
**架构：** 简洁两页方案 - 历史列表页 + 图表详情页

---

## 1. 背景与目标

### 1.1 背景

当前小程序首页展示了 4 个功能模块（数据处理、统计分析、数据画图、实用工具），但这些功能卡片没有实际的跳转逻辑，不符合微信小程序审核规范：

- **违反 3.1.5**：功能过于简单，卡片无法点击
- **违反 3.1.2**：首页无法直达或 2 次点击到达服务类目页面
- **违反 3.3.1**：提交的小程序不是完成品

### 1.2 目标

1. 实现"数据画图"功能，用户可以查看历史分析记录
2. 为"开发中"的功能添加明确标识，避免误导用户
3. 符合微信小程序审核规范，确保审核通过

### 1.3 功能范围

**本期实现：**
- 数据画图历史记录列表（分页加载）
- 图表详情查看（图片预览、保存）
- 首页功能卡片点击事件
- 开发中功能标记

**暂不实现：**
- 数据处理功能（开发中）
- 统计分析功能（开发中）
- 实用工具功能（开发中）

## 3. API 设计

### 3.1 API 封装

在 `utils/api.js` 中添加数据画图相关的 API：

```javascript
/**
 * 数据画图相关API
 */
const chartApi = {
  /**
   * 获取历史记录列表（分页）
   * @param {Object} params - { page: { page: 0, pageSize: 10, sort: ['id,desc'] } }
   */
  getHistoryList(params) {
    return request.post("/api/userDraw/page_list", params);
  },

  /**
   * 删除历史记录
   * @param {Array} ids - 记录ID数组
   */
  deleteHistory(ids) {
    return request.post("/api/userDraw/del", ids);
  }
};

module.exports = {
  authApi,
  chartApi,  // 新增
  request,
  login,
};
```

### 3.2 数据结构

**请求参数：**

```javascript
{
  page: {
    page: 0,        // 页码（从0开始）
    pageSize: 10,   // 每页数量
    sort: ['id,desc'] // 排序规则
  }
}
```

**响应数据：**

```javascript
{
  code: 200,
  data: {
    content: [      // 历史记录数组
      {
        id: 1,
        image: "https://...",  // 图片URL
        createTime: "2026-03-19 10:00:00",
        updateTime: "2026-03-19 10:00:00"
      }
    ],
    totalElements: 50  // 总记录数
  }
}
```

### 3.3 Token 处理

Token 已在 `utils/request.js` 的 `buildHeaders` 方法中统一处理：

```javascript
buildHeaders(headers = {}) {
  const token = this.getToken();
  if (token) {
    defaultHeaders["Authorization"] = `Bearer ${token}`;
  }
  return { ...defaultHeaders, ...headers };
}
```

API 调用时无需额外处理 token。

### 4.2 历史记录列表页（pages/chart-history/index）

#### 4.2.1 功能特性

1. **分页加载**：每页 10 条记录
2. **下拉刷新**：重新加载第一页数据
3. **上拉加载更多**：自动加载下一页
4. **空状态提示**：无数据时显示友好提示
5. **加载状态**：显示加载中、没有更多等状态

#### 4.2.2 WXML 结构

```xml
<view class="container">
  <view class="header">
    <text class="title">历史记录</text>
  </view>

  <scroll-view
    scroll-y
    class="history-list"
    bindscrolltolower="onReachBottom"
    refresher-enabled="{{true}}"
    refresher-triggered="{{refreshing}}"
    bindrefresherrefresh="onPullDownRefresh"
  >
    <view class="list-content">
      <!-- 历史记录列表 -->
      <view
        wx:for="{{historyList}}"
        wx:key="id"
        class="history-item"
        bindtap="viewDetail"
        data-id="{{item.id}}"
        data-image="{{item.image}}"
      >
        <view class="item-image">
          <image src="{{item.image}}" mode="aspectFill" />
        </view>
        <view class="item-info">
          <text class="create-time">创建时间：{{item.createTime}}</text>
          <text class="update-time">修改时间：{{item.updateTime}}</text>
        </view>
        <view class="item-actions">
          <view class="arrow">›</view>
        </view>
      </view>

      <!-- 加载状态 -->
      <view class="loading-more" wx:if="{{loading}}">
        <text>加载中...</text>
      </view>

      <!-- 没有更多数据 -->
      <view class="no-more" wx:if="{{noMore && historyList.length > 0}}">
        <text>没有更多了</text>
      </view>

      <!-- 空状态 -->
      <view class="empty-state" wx:if="{{historyList.length === 0 && !loading}}">
        <text class="empty-icon">📊</text>
        <text class="empty-text">暂无历史记录</text>
      </view>
    </view>
  </scroll-view>
</view>
```

#### 4.2.3 JS 逻辑

```javascript
const { chartApi } = require("../../utils/api");

Page({
  data: {
    historyList: [],      // 历史记录列表
    loading: false,       // 加载状态
    refreshing: false,    // 刷新状态
    noMore: false,        // 是否没有更多数据
    page: 0,              // 当前页码
    pageSize: 10,         // 每页数量
    totalElements: 0      // 总记录数
  },

  onLoad: function (options) {
    this.loadHistoryList();
  },

  // 加载历史记录列表
  loadHistoryList: function(isRefresh = false) {
    if (this.data.loading) return;

    // 如果是刷新，重置页码
    if (isRefresh) {
      this.setData({
        page: 0,
        historyList: [],
        noMore: false
      });
    }

    this.setData({ loading: true });

    chartApi.getHistoryList({
      page: {
        page: this.data.page,
        pageSize: this.data.pageSize,
        sort: ['id,desc']
      }
    }).then(res => {
      if (res.code === 200) {
        const newList = isRefresh ? res.data.content : [...this.data.historyList, ...res.data.content];
        const noMore = newList.length >= res.data.totalElements;

        this.setData({
          historyList: newList,
          totalElements: res.data.totalElements,
          loading: false,
          refreshing: false,
          noMore: noMore
        });
      } else {
        this.showError(res.message || '加载失败');
      }
    }).catch(err => {
      this.showError('网络错误，请稍后重试');
    });
  },

  // 下拉刷新
  onPullDownRefresh: function() {
    this.setData({ refreshing: true });
    this.loadHistoryList(true);
  },

  // 上拉加载更多
  onReachBottom: function() {
    if (this.data.noMore || this.data.loading) return;

    this.setData({
      page: this.data.page + 1
    });
    this.loadHistoryList();
  },

  // 查看详情
  viewDetail: function(e) {
    const { id, image } = e.currentTarget.dataset;
    wx.navigateTo({
      url: `/pages/chart-detail/index?id=${id}&image=${encodeURIComponent(image)}`
    });
  },

  // 显示错误提示
  showError: function(message) {
    this.setData({
      loading: false,
      refreshing: false
    });
    wx.showToast({
      title: message,
      icon: 'none',
      duration: 2000
    });
  }
});
```

### 4.3 图表详情页（pages/chart-detail/index）

#### 4.3.1 功能特性

1. **图片显示**：显示完整的图表图片
2. **图片预览**：点击图片或预览按钮可全屏预览
3. **保存图片**：下载图片并保存到相册
4. **权限处理**：保存图片需要相册权限，拒绝时引导用户授权

#### 4.3.2 WXML 结构

```xml
<view class="container">
  <view class="image-container">
    <image
      src="{{imageUrl}}"
      mode="widthFix"
      class="chart-image"
      bindtap="previewImage"
      show-menu-by-longpress="{{true}}"
    />
  </view>

  <view class="actions">
    <button class="action-btn" bindtap="saveImage">
      <text class="icon">💾</text>
      <text>保存图片</text>
    </button>
    <button class="action-btn" bindtap="previewImage">
      <text class="icon">🔍</text>
      <text>预览</text>
    </button>
  </view>
</view>
```

#### 4.3.3 JS 逻辑

```javascript
Page({
  data: {
    id: '',
    imageUrl: ''
  },

  onLoad: function (options) {
    const { id, image } = options;
    this.setData({
      id: id,
      imageUrl: decodeURIComponent(image)
    });
  },

  // 预览图片
  previewImage: function() {
    wx.previewImage({
      urls: [this.data.imageUrl],
      current: this.data.imageUrl
    });
  },

  // 保存图片到相册
  saveImage: function() {
    wx.showLoading({
      title: '保存中...',
      mask: true
    });

    // 先下载图片
    wx.downloadFile({
      url: this.data.imageUrl,
      success: (res) => {
        if (res.statusCode === 200) {
          // 保存到相册
          wx.saveImageToPhotosAlbum({
            filePath: res.tempFilePath,
            success: () => {
              wx.hideLoading();
              wx.showToast({
                title: '保存成功',
                icon: 'success',
                duration: 2000
              });
            },
            fail: (err) => {
              wx.hideLoading();
              if (err.errMsg.includes('auth deny')) {
                // 用户拒绝授权
                wx.showModal({
                  title: '提示',
                  content: '需要您授权保存图片到相册',
                  confirmText: '去授权',
                  success: (modalRes) => {
                    if (modalRes.confirm) {
                      wx.openSetting();
                    }
                  }
                });
              } else {
                wx.showToast({
                  title: '保存失败',
                  icon: 'none',
                  duration: 2000
                });
              }
            }
          });
        } else {
          wx.hideLoading();
          wx.showToast({
            title: '下载失败',
            icon: 'none',
            duration: 2000
          });
        }
      },
      fail: () => {
        wx.hideLoading();
        wx.showToast({
          title: '下载失败',
          icon: 'none',
          duration: 2000
        });
      }
    });
  }
});
```

## 5. 错误处理与用户体验

### 5.1 错误处理策略

| 错误类型 | 处理方式 |
|---------|---------|
| 网络请求失败 | 显示 Toast 提示，提供下拉刷新重试 |
| 无历史记录 | 显示空状态提示（图标 + 文字） |
| 图片加载失败 | 显示占位图或错误提示 |
| 保存图片失败 | 显示具体错误原因（网络/权限） |
| 权限被拒绝 | 弹窗引导用户去设置页面授权 |

### 5.2 用户体验优化

#### 5.2.1 图片优化

- **列表缩略图**：使用 `mode="aspectFill"` 保持比例裁剪
- **详情页原图**：使用 `mode="widthFix"` 保持宽度自适应高度
- **长按保存**：设置 `show-menu-by-longpress="{{true}}"` 支持长按菜单

#### 5.2.2 交互反馈

- **卡片点击**：添加 `hover-class="card-hover"` 提供视觉反馈
- **操作提示**：所有操作都有 Toast 或 Modal 反馈
- **加载状态**：明确显示"加载中"、"没有更多"等状态

#### 5.2.3 性能优化

- **分页加载**：每页 10 条，避免一次加载过多数据
- **列表优化**：使用 `wx:key="id"` 优化渲染性能
- **图片懒加载**：scroll-view 自动实现懒加载

## 7. 技术实现要点

### 7.1 关键技术

1. **scroll-view 下拉刷新**：使用 `refresher-enabled` 和 `bindrefresherrefresh`
2. **上拉加载更多**：监听 `bindscrolltolower` 事件
3. **图片预览**：使用 `wx.previewImage` API
4. **图片保存**：使用 `wx.downloadFile` + `wx.saveImageToPhotosAlbum`
5. **权限处理**：使用 `wx.openSetting` 引导授权

### 7.2 数据流

```
用户操作 → 页面事件 → API 调用 → 后端处理 → 返回数据 → 更新页面
```

### 7.3 状态管理

- **loading**：控制加载状态显示
- **refreshing**：控制下拉刷新动画
- **noMore**：控制"没有更多"提示
- **page**：当前页码，用于分页加载
