# 深色模式实现设计文档

> **创建日期：** 2026-03-17
> **状态：** 设计阶段
> **实施方案：** CSS 变量 + 媒体查询

## 目标

为 goto-web 项目实现深色模式支持，使页面主题能够自动跟随系统主题，同时提供手动切换功能。

## 需求概述

### 功能需求

1. **自动主题切换**：默认跟随操作系统的深色/浅色模式设置
2. **手动主题切换**：用户可以通过顶部导航栏的按钮手动切换主题
3. **主题持久化**：用户的主题偏好保存到 localStorage
4. **平滑过渡**：主题切换时使用 0.3 秒的平滑渐变动画
5. **三种模式**：
   - `auto`：自动跟随系统主题（默认）
   - `light`：强制浅色模式
   - `dark`：强制深色模式

### 非功能需求

1. **性能**：纯 CSS 实现，无 JS 运行时开销
2. **兼容性**：支持现代浏览器（Chrome 76+, Firefox 67+, Safari 12.1+）
3. **可维护性**：基于现有 CSS 变量系统，集中管理颜色
4. **渐进增强**：旧浏览器降级到浅色模式

## 设计方案

### 方案选择

**选定方案：CSS 变量 + 媒体查询**

**理由：**
- 基于现有的 `theme-variables.scss`，改动最小
- 性能最优，纯 CSS 实现
- 易于维护，变量集中管理
- 与现有架构无缝集成

#### 优先级逻辑

1. 如果用户手动设置了主题（`data-theme="light"` 或 `"dark"`），使用用户偏好
2. 否则，跟随系统主题（`prefers-color-scheme`）
3. 默认使用浅色模式

### 组件设计

#### ThemeToggle 组件

**位置：** `src/views/goto/components/ThemeToggle.vue`

**功能：**
- 显示当前主题图标（太阳/月亮/显示器）
- 点击循环切换主题：auto → light → dark → auto
- 响应式更新图标

**实现：**

```vue
<template>
  <div class="theme-toggle" @click="toggleTheme" :title="themeTooltip">
    <i :class="themeIcon"></i>
  </div>
</template>

<script>
export default {
  name: 'ThemeToggle',
  computed: {
    currentTheme() {
      return this.$store.state.settings.theme || 'auto'
    },
    themeIcon() {
      if (this.currentTheme === 'dark') {
        return 'el-icon-moon'
      } else if (this.currentTheme === 'light') {
        return 'el-icon-sunny'
      } else {
        return 'el-icon-monitor'
      }
    },
    themeTooltip() {
      const tooltips = {
        auto: '自动跟随系统',
        light: '浅色模式',
        dark: '深色模式'
      }
      return tooltips[this.currentTheme]
    }
  },
  methods: {
    toggleTheme() {
      const themes = ['auto', 'light', 'dark']
      const currentIndex = themes.indexOf(this.currentTheme)
      const nextTheme = themes[(currentIndex + 1) % themes.length]
      this.$store.dispatch('settings/setTheme', nextTheme)
    }
  }
}
</script>

<style lang="scss" scoped>
.theme-toggle {
  display: flex;
  align-items: center;
  justify-content: center;
  width: 40px;
  height: 40px;
  cursor: pointer;
  border-radius: 8px;
  transition: background-color 0.2s;

  &:hover {
    background-color: var(--hover-bg);
  }

  i {
    font-size: 20px;
    color: var(--text-main);
  }
}
</style>
```

#### 主题工具类

**位置：** `src/utils/theme.js`

**功能：**
- 初始化主题
- 应用主题到 DOM
- 获取当前实际显示的主题
- 监听系统主题变化

**实现：**

```javascript
// src/utils/theme.js
export const ThemeManager = {
  /**
   * 初始化主题
   * 在应用启动时调用（main.js）
   */
  init() {
    const savedTheme = localStorage.getItem('theme') || 'auto'
    this.applyTheme(savedTheme)
    this.listenSystemThemeChange()
  },

  /**
   * 应用主题到 DOM
   * @param {string} theme - 'auto' | 'light' | 'dark'
   */
  applyTheme(theme) {
    const root = document.documentElement

    if (theme === 'auto') {
      // 移除 data-theme，让媒体查询生效
      root.removeAttribute('data-theme')
    } else {
      // 设置 data-theme 属性
      root.setAttribute('data-theme', theme)
    }

    // 保存到 localStorage
    try {
      localStorage.setItem('theme', theme)
    } catch (e) {
      console.warn('无法保存主题偏好到 localStorage:', e)
    }
  },

  /**
   * 获取当前实际显示的主题（考虑系统偏好）
   * @returns {string} 'light' | 'dark'
   */
  getCurrentTheme() {
    const savedTheme = localStorage.getItem('theme') || 'auto'

    if (savedTheme === 'auto') {
      // 检测系统主题
      return window.matchMedia('(prefers-color-scheme: dark)').matches
        ? 'dark'
        : 'light'
    }

    return savedTheme
  },

  /**
   * 监听系统主题变化
   */
  listenSystemThemeChange() {
    const mediaQuery = window.matchMedia('(prefers-color-scheme: dark)')

    mediaQuery.addEventListener('change', (e) => {
      const savedTheme = localStorage.getItem('theme') || 'auto'
      // 只有在 auto 模式下才响应系统主题变化
      if (savedTheme === 'auto') {
        console.log('系统主题已变化:', e.matches ? 'dark' : 'light')
        // CSS 会自动响应，无需手动操作
      }
    })
  },

  /**
   * 检查浏览器是否支持 CSS 变量
   * @returns {boolean}
   */
  isCSSVariableSupported() {
    return window.CSS && window.CSS.supports &&
           window.CSS.supports('color', 'var(--test)')
  }
}
```

### 数据流设计

#### 主题切换流程

```
用户点击主题切换按钮
    ↓
ThemeToggle 组件触发 toggleTheme()
    ↓
计算下一个主题（auto → light → dark → auto）
    ↓
调用 Vuex action: settings/setTheme(nextTheme)
    ↓
更新 Vuex state.settings.theme
    ↓
调用 ThemeManager.applyTheme(nextTheme)
    ↓
根据主题值设置/移除 data-theme 属性
    ↓
保存到 localStorage
    ↓
CSS 变量自动响应（通过选择器优先级）
    ↓
过渡动画生效（0.3秒平滑渐变）
    ↓
页面完成主题切换
```

#### 初始化流程

```
应用启动（main.js）
    ↓
调用 ThemeManager.init()
    ↓
从 localStorage 读取保存的主题偏好
    ↓
如果没有保存，默认使用 'auto'
    ↓
应用主题到 document.documentElement
    ↓
启动系统主题变化监听
    ↓
Vuex store 同步主题状态
    ↓
页面以正确的主题渲染
```

### Vuex 集成

在 `src/store/modules/settings.js` 中添加：

```javascript
import { ThemeManager } from '@/utils/theme'

const state = {
  theme: localStorage.getItem('theme') || 'auto'
}

const mutations = {
  SET_THEME: (state, theme) => {
    state.theme = theme
  }
}

const actions = {
  setTheme({ commit }, theme) {
    commit('SET_THEME', theme)
    ThemeManager.applyTheme(theme)
  }
}

export default {
  namespaced: true,
  state,
  mutations,
  actions
}
```

## 错误处理

### 1. localStorage 不可用

**场景：** 隐私模式、存储配额已满、浏览器禁用

**处理：**
```javascript
try {
  localStorage.setItem('theme', theme)
} catch (e) {
  console.warn('无法保存主题偏好:', e)
  // 降级：仅使用内存状态，不持久化
}
```

### 2. CSS 变量不支持

**场景：** IE11 等旧浏览器

**处理：**
```javascript
if (!ThemeManager.isCSSVariableSupported()) {
  console.warn('浏览器不支持 CSS 变量，使用默认浅色主题')
  // 降级：使用固定的浅色主题
  return
}
```

### 3. 主题切换卡顿

**场景：** 大量 DOM 元素导致重绘性能问题

**优化：**
- 使用 `will-change` CSS 属性
- 限制过渡动画的属性范围
- 使用 `requestAnimationFrame` 优化 JS 操作
