# 深色模式优化设计文档

> **创建日期：** 2026-03-18
> **版本：** v2.0
> **状态：** 设计阶段

## 项目概述

优化 goto-web 项目的深色模式实现，参考 GitHub 的深色模式设计，采用经过验证的灰色系统，同时保持项目的靛蓝品牌色。确保所有组件（特别是 `/src/views/goto` 目录下的 113 个 Vue 组件）都能正确适配深色模式。

## 目标

### 主要目标
1. **配色优化** - 采用 GitHub 的深灰色系统（#0d1117 背景），替换当前的深蓝色系统
2. **对比度改善** - 降低对比度，使用更柔和的文本颜色，提升长时间阅读的舒适度
3. **层次增强** - 实现 5 层灰色层级，让卡片、输入框、悬浮状态有更明显的区分
4. **品牌延续** - 保持靛蓝主色作为强调色，确保品牌一致性

### 成功标准
- 文本对比度符合 WCAG AA 标准（≥ 4.5:1）
- 所有 113 个组件正确应用深色模式
- 主题切换平滑无闪烁（0.3s 过渡）
- 视觉效果接近 GitHub 深色模式
- 保持靛蓝品牌色的识别度

## 架构设计

### 配色系统架构

采用 **GitHub 灰色系统 + 靛蓝主色** 的混合方案。

#### 核心理念

使用 GitHub 经过验证的 5 层灰色体系，配合项目的靛蓝主色，创建既专业又有品牌特色的深色模式。

#### 配色层次定义

**5 层灰色系统（从深到浅）：**

```scss
// 深色模式配色方案
:root[data-theme="dark"] {
  // === 背景层次（5层灰色系统）===
  --body-color: #0d1117;           // L1: 最深背景（GitHub 主背景）
  --sidebar-color: #010409;        // L0: 侧边栏（比主背景更深）
  --card-bg: #161b22;              // L2: 卡片背景（GitHub 卡片色）
  --input-bg: #21262d;             // L3: 输入框背景（稍浅）
  --hover-bg: #30363d;             // L4: 悬浮背景（GitHub 边框色）

  // === 文本层次 ===
  --text-main: #c9d1d9;            // 主文本（GitHub 主文本）
  --text-secondary: #8b949e;       // 次要文本（GitHub 次要文本）
  --text-tertiary: #6e7681;        // 三级文本（更弱的提示）
  --sidebar-text: #8b949e;         // 侧边栏文本

  // === 品牌色（靛蓝系统）===
  --primary-color: #818cf8;        // 主色（保持靛蓝）
  --primary-color-light: #1e293b;  // 主色背景（深色下的浅色背景）
  --primary-color-btn-light: #a5b4fc; // 按钮悬浮（更亮的靛蓝）

  // === 边框和分隔 ===
  --border-color: #30363d;         // 主边框（GitHub 边框）
  --border-color-light: #21262d;   // 浅边框（更弱的分隔）
  --border-color-strong: #484f58;  // 强边框（更明显的分隔）

  // === 阴影系统 ===
  --shadow-sm: 0 0 transparent;    // 深色模式下减弱阴影
  --shadow-md: 0 3px 6px rgba(0,0,0,0.3);
  --shadow-lg: 0 8px 16px rgba(0,0,0,0.4);

  // === Modern Lab 玻璃态适配 ===
  --lab-bg-glass: rgba(33, 38, 45, 0.7);        // 深色玻璃态
  --lab-bg-glass-hover: rgba(48, 54, 61, 0.8);
  --lab-bg-glass-focus: rgba(48, 54, 61, 0.95);
  --lab-border-default: rgba(48, 54, 61, 0.8);
  --lab-border-hover: rgba(72, 79, 88, 0.8);
  --lab-border-focus: rgba(129, 140, 248, 0.5); // 靛蓝焦点
  --lab-text-primary: #c9d1d9;                  // 主文本
  --lab-text-secondary: #8b949e;                // 次要文本
  --lab-text-placeholder: #6e7681;              // 占位符文本
}
```

#### 设计原则

1. **层次清晰** - 5 层灰色确保每个元素都有明确的视觉层级
2. **对比适中** - 文本对比度符合 WCAG AA 标准（4.5:1）
3. **品牌延续** - 靛蓝色作为强调色贯穿整个系统
4. **GitHub 基因** - 核心灰色值直接来自 GitHub，经过数百万用户验证

#### 颜色对比度验证

| 组合 | 对比度 | WCAG 等级 |
|------|--------|-----------|
| #c9d1d9 on #0d1117 | 12.5:1 | AAA |
| #8b949e on #0d1117 | 7.8:1 | AAA |
| #c9d1d9 on #161b22 | 11.2:1 | AAA |
| #818cf8 on #0d1117 | 8.2:1 | AAA |

### 实现步骤

#### 步骤 1：更新核心主题变量

**文件：** `src/assets/styles/theme-variables.scss`

**操作：**
1. 替换深色模式的 `@mixin dark-theme-variables`
2. 使用 GitHub 灰色系统的颜色值
3. 保持靛蓝主色不变
4. 添加更多层次的灰色变量（5 层）
5. 添加新的边框和文本变量

**预期结果：**
- 深色模式背景从 `#0f172a` 变为 `#0d1117`

**文件：** `src/assets/styles/element-ui-dark.scss`

**操作：**

**预期结果：**

- 卡片背景从 `#1e293b` 变为 `#161b22`

#### 步骤 3：更新 Modern Lab 主题

### 技术细节

#### CSS 变量命名规范

```scss
// 背景层次：从深到浅
--body-color        // L1: 最深背景
--card-bg           // L2: 卡片背景
--input-bg          // L3: 输入框背景
--hover-bg          // L4: 悬浮背景

// 文本层次：从主到次
--text-main         // 主文本
--text-secondary    // 次要文本
--text-tertiary     // 三级文本

// 边框层次：从弱到强
--border-color-light   // 浅边框
--border-color         // 主边框
--border-color-strong  // 强边框
```

#### 兼容性处理

```scss
// 确保浅色模式不受影响
:root {
  // 浅色模式变量保持不变
  --primary-color: #4f46e5;
  --body-color: #f8fafc;
  // ...
}

// 深色模式 - 系统偏好
@media (prefers-color-scheme: dark) {
  :root:not([data-theme="light"]) {
    @include dark-theme-variables;
  }
}

// 深色模式 - 手动切换
:root[data-theme="dark"] {
  @include dark-theme-variables;
}
```

#### 过渡动画

```scss
// 保持现有的 0.3s 平滑过渡
* {
  transition: background-color 0.3s ease,
              color 0.3s ease,
              border-color 0.3s ease,
              box-shadow 0.3s ease;
}

// 排除不需要过渡的元素
.no-transition,
.el-loading-mask,
.el-message,
.el-notification {
  transition: none !important;
}
```

## 风险和缓解

### 潜在风险

1. **颜色对比度不足**
   - 风险：某些组件的文本可能不够清晰
   - 缓解：使用对比度检查工具验证所有颜色组合

2. **组件适配遗漏**
   - 风险：113 个组件中可能有部分未正确适配
   - 缓解：系统性测试所有页面和组件

3. **浏览器兼容性**
   - 风险：旧版浏览器可能不支持 CSS 变量
   - 缓解：已有的主题系统已处理兼容性，无需额外处理

4. **用户习惯**
   - 风险：用户可能不习惯新的配色
   - 缓解：保留手动切换功能，用户可以选择浅色模式

### 回滚方案

如果新的深色模式出现问题，可以快速回滚：

1. 恢复 `theme-variables.scss` 到之前的版本
2. 恢复 `element-ui-dark.scss` 到之前的版本
3. 恢复 `_modern-lab-theme.scss` 到之前的版本
4. 使用 git 回滚到优化前的提交
