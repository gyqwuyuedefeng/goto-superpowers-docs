# Excel 格式转换页面控制台清理设计

## 背景

`src/views/goto/toolsBox/type/excelFormatConvert.vue` 的引导图弹窗在模板中绑定了 `@tab-click="onGuideTabChange"`，但组件 `methods` 未定义该方法。Vue 2 在组件首次渲染及文件选择、解析、上传等状态更新触发重渲染时，都会报告属性或方法未定义告警。

该问题由提交 `9cc17e6` 引入：提交为工具页增加了双 Tab 引导图模板、`guideActiveTab` 状态和基于 Tab 选择图片的计算属性，但遗漏了模板所引用的事件处理方法。同仓库组件 `src/views/goto/plot/type/fileMark/formatConvertDialog.vue` 已提供行为完整的参考实现。

目标组件另有 17 处 `console.log`，覆盖文件选择、上传、转换任务提交、WebSocket 状态和结果下载等正常业务流程。这些输出属于调试日志，会干扰开发者定位真正的告警和异常。

## 目标

1. 消除 `onGuideTabChange` 未定义产生的 Vue 告警。
2. 保持引导图两个 Tab 的切换行为和图片错误状态恢复能力。
3. 删除目标组件全部 17 处 `console.log` 调试输出。
4. 保留目标组件现有 4 处 `console.error` 异常日志。
5. 不改变文件上传、Excel 解析、转换任务、WebSocket、下载、Toast 或页面状态行为。
6. 增加能够阻止同类回归的定向测试。

## 非目标

1. 不处理 SockJS/Webpack Dev Server 产生的 Permissions Policy `unload` 告警。
2. 不处理 HMR 等开发服务器状态信息。
3. 不处理 Chrome 扩展或浏览器内置 AI 输出的信息。
4. 不清理 `goto-web` 其他组件中的控制台输出。
5. 不升级 Vue、Element UI、Webpack 或其他前端依赖。
6. 不重构引导图弹窗、Excel 转换流程或 WebSocket 通知架构。

## 方案选择

### 采用方案：补齐事件方法并清理调试日志

在目标组件中新增 `onGuideTabChange(tab)`，沿用同仓库参考组件的行为：将 `tab.name` 写入 `guideActiveTab`，并将 `guideImageLoadError` 重置为 `false`。同时逐项删除目标组件中的 `console.log`，不改动其所在分支的其他语句。

该方案直接修复模板与组件实例之间的契约缺失，改动范围最小，并与现有同类组件保持一致。

### 未采用方案一：使用 watch 监听 Tab

可以移除 `@tab-click`，再监听 `guideActiveTab` 来重置图片错误状态。该方案能实现相同行为，但会把显式的用户事件处理改为隐式状态监听，增加理解和测试成本。

### 未采用方案二：只移除事件绑定

仅删除 `@tab-click` 可以消除 Vue 告警，因为 `v-model` 仍会更新 Tab 值。但切换 Tab 后旧的图片加载失败状态不会清除，新 Tab 的图片将没有重新加载机会，因此会造成功能退化。

## 修改范围

| 文件 | 修改内容 |
| --- | --- |
| `goto-web/src/views/goto/toolsBox/type/excelFormatConvert.vue` | 补齐 Tab 切换方法；删除 17 处 `console.log`；保留 `console.error` |

## 组件行为与数据流

1. 用户点击页面右上角的引导入口，`openGuideDialog` 清除图片错误状态并打开弹窗。
2. 用户切换 Element UI Tab，`el-tabs` 通过 `tab-click` 将目标 Tab 实例传给 `onGuideTabChange`。
3. `onGuideTabChange` 将 `tab.name` 同步到 `guideActiveTab`。
4. `currentGuideImage` 根据 `guideActiveTab` 返回宽转长或长转宽 SVG。
5. `onGuideTabChange` 同时把 `guideImageLoadError` 重置为 `false`，防止前一个 Tab 的加载失败状态污染新 Tab。
6. 如果新图片仍加载失败，现有 `onGuideImageError` 再将错误状态设为 `true`，模板显示已有兜底文案。

## 日志清理规则

删除所有 `console.log`，包括以下正常流程信息：

- 文件选择、上传成功和移除；
- 转换请求参数、任务提交和处理状态；
- WebSocket 消息、UUID 过滤和任务状态；
- taskId 提取及兜底；
- 转换完成和下载按钮状态。

保留所有 `console.error`，继续覆盖：

- 文件处理异常；
- 转换异常；
- 下载异常；
- 缺少可下载文件的异常状态。

删除日志时只移除对应调用语句，不合并条件分支，不更改状态赋值、Toast、返回值、异步等待或错误传播。

## 错误处理

本次不增加新的错误类型或用户提示。图片加载失败继续由 `onGuideImageError` 和模板兜底文案处理；文件、转换和下载异常继续使用现有 `console.error`、Toast 及页面错误状态。

`onGuideTabChange` 不增加对空 Tab 的防御逻辑，因为该方法只由 Element UI `tab-click` 事件调用，事件契约保证提供包含 `name` 的 Tab 实例。增加额外兜底会扩大本次修改范围且掩盖组件契约错误。

## 测试设计

### Tab 切换行为测试

在现有组件配置对象测试中构造 `guideImageLoadError = true` 的实例，调用：

```js
onGuideTabChange({ name: 'l2w' })
```

断言：

- `guideActiveTab` 更新为 `l2w`；
- `guideImageLoadError` 更新为 `false`。

该测试直接证明模板引用的方法存在，并覆盖本次缺失的方法行为。

### 日志约束测试

读取目标 Vue 源码并断言不存在 `console.log(`。测试不禁止 `console.error`，以免破坏已确认需要保留的异常诊断能力。

### 既有回归测试

继续执行目标组件现有 5 个测试，覆盖：

- 文件就绪时允许无固定列的宽转长转换；
- 空固定列转换为后端兼容的 `null`；
- 三类列选择状态相互隔离；
- 切换转换类型时清理旧状态；
- 重置表单时保留当前转换类型。

### 静态检查与页面验收

1. 对修改的 Vue 文件和测试文件执行 ESLint。
2. 打开引导弹窗，依次切换宽转长和长转宽 Tab，确认图片正确切换。
3. 选择一个合法 Excel 文件，覆盖解析和上传引发的组件重渲染。
4. 确认控制台不再出现 `onGuideTabChange` 未定义告警，也不再出现目标组件业务调试日志。
5. SockJS、HMR 和浏览器扩展输出不计入本次验收结果。

## 验收标准

1. 页面首次渲染、文件选择、Excel 解析及上传后的重渲染均不再报告 `onGuideTabChange` 未定义。
2. 两个引导图 Tab 可以正常切换，切换时会清除前一个 Tab 的图片加载错误状态。
3. 目标组件源码不存在 `console.log`。
4. 目标组件现有 `console.error` 保持不变。
5. 文件上传、转换任务、WebSocket 状态、结果下载和用户 Toast 行为保持不变。
6. 新增定向测试、现有目标组件测试和修改文件 ESLint 全部通过。
