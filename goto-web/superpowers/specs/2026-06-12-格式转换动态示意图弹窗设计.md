# 格式转换动态示意图弹窗设计

## 背景

目标组件为 `goto-web/src/views/goto/plot/type/fileMark/formatConvertDialog.vue`。用户提供的两个路径实际指向同一个 Vue 组件，因此本次设计面向该复用组件生效；凡是打开该组件的业务入口，都会展示同一套示意图入口和弹窗行为。

现有组件用于 Excel 格式转换，支持“宽转长”和“长转宽”两种转换类型。页面底部已有文字说明，但用户希望在合适位置增加一个美观图标，点击后通过弹窗查看对应的动态 SVG 示意图：

- `tmp/宽格式转长格式-动态说明.svg`
- `tmp/长格式转宽格式-动态说明.svg`

## 已确认方案

采用 A 方案：在“转换类型”单选区域右侧增加一个圆形说明图标入口。

选择该位置的原因：

- 入口贴近“宽转长 / 长转宽”的决策点，用户切换类型时最容易发现。
- 不占用具体参数区域，避免误以为示意图只解释某个参数。
- 与现有禁用提示 `el-tooltip` 所在区域一致，视觉语义自然。

点击图标后打开示意图弹窗。弹窗必须相对当前视口水平、垂直居中显示。

## 交互设计

转换类型区域增加一个说明按钮：

- 默认显示为小尺寸圆形图标按钮，不额外展示“查看示意图”等文字，避免挤占单选区域。
- 鼠标悬停显示 tooltip：`查看当前转换类型的动态示意图`。
- 点击图标时，根据当前 `formData.type` 打开对应示意图弹窗。

示意图选择规则：

- `formData.type === 'w2l'`：显示“宽格式转长格式-动态说明.svg”。
- `formData.type === 'l2w'`：显示“长格式转宽格式-动态说明.svg”。
- 当 `params.scenario === 'handsontable'` 导致“长转宽”禁用时，用户只能停留在“宽转长”，图标点击后只展示宽转长示意图。

弹窗行为：

- 使用遮罩层，避免用户误操作底层转换表单。
- 标题根据当前类型显示：`宽转长动态示意图` 或 `长转宽动态示意图`。
- 弹窗支持右上角关闭按钮，关闭后不修改任何表单数据。
- 弹窗内容区域展示 SVG，图片按容器宽度自适应；当 SVG 高度超出可视区域时，内容区滚动而不是让弹窗溢出屏幕。
- 弹窗需要垂直居中，不能沿用默认靠上的 `el-dialog` 位置。

## 资源设计

实现时不直接引用 `tmp` 目录。`tmp` 只作为用户提供的源文件位置，正式资源应复制到前端源码目录，例如：

```text
goto-web/src/assets/goto/plot/format-convert/wide-to-long-guide.svg
goto-web/src/assets/goto/plot/format-convert/long-to-wide-guide.svg
```

组件通过 `import` 引入资源，避免运行时路径依赖临时目录，也确保生产构建能正确打包。

## 组件结构

本次改动保持在 `formatConvertDialog.vue` 内部完成，不引入新的业务组件，避免扩大影响面。

新增状态：

- `guideDialogVisible`：控制示意图弹窗显示。

新增计算属性：

- `currentGuideTitle`：根据转换类型返回弹窗标题。
- `currentGuideImage`：根据转换类型返回当前 SVG 资源。

新增方法：

- `openGuideDialog()`：打开示意图弹窗。
- `closeGuideDialog()`：关闭示意图弹窗。

模板新增位置：

- 在“转换类型”单选组右侧插入 `el-tooltip + el-button` 图标按钮。
- 在主 `Dialog` 的 `body` 插槽内追加一个独立 `el-dialog`，用于展示示意图。

## 样式设计

图标入口样式：

- 小尺寸圆形按钮，使用浅蓝背景、蓝色图标、细边框。
- 与单选项保持同一行垂直居中。
- 不破坏 `el-form-item` 的标签宽度和现有表单排版。

弹窗样式：

- 使用自定义 `custom-class` 控制示意图弹窗。
- 通过 `top="50%"` 配合 `transform: translateY(-50%)` 和 `margin-top: 0` 实现垂直居中。
- 弹窗宽度按桌面端设置为较宽展示区域，并通过 `max-width` 适配小屏。
- 图片使用 `max-width: 100%`，内容区域使用 `max-height: calc(100vh - 96px)` 和 `overflow: auto`。

## 数据流

1. 用户打开格式转换弹窗。
2. 用户选择或保持当前转换类型。
3. 用户点击“查看示意图”图标。
4. 组件读取 `formData.type`。
5. 组件计算标题和 SVG 地址。
6. 示意图弹窗垂直居中打开。
7. 用户关闭示意图弹窗，表单状态保持不变。

## 边界与错误处理

- 如果当前类型为空或异常，默认按 `w2l` 展示宽转长示意图，避免空白弹窗。
- 如果图片加载失败，应显示简短错误占位文案，例如 `示意图加载失败，请稍后重试`。
- 示意图弹窗关闭不触发主转换弹窗的 `close`、`confirmFun` 或表单重置逻辑。
- 切换转换类型时，如果示意图弹窗已经打开，弹窗内容应随当前类型计算结果更新；无需强制关闭。

## 测试与验收

手工验收：

- 打开格式转换弹窗，确认说明图标位于“转换类型”单选项右侧。
- 选择“宽转长”后点击图标，确认弹窗标题和 SVG 均为宽转长示意图。
- 在允许“长转宽”的工具场景下，选择“长转宽”后点击图标，确认弹窗标题和 SVG 均为长转宽示意图。
- 在 Handsontable 场景下确认“长转宽”仍禁用，点击图标只展示宽转长示意图。
- 确认示意图弹窗水平、垂直居中，关闭弹窗不影响主表单内容。
- 缩小浏览器窗口后确认弹窗不溢出屏幕，示意图区域可滚动查看。

代码校验：

- 运行前端 lint，确认 Vue 模板、脚本和样式无格式或语法问题。
- 如项目当前 lint 存在历史问题，应至少确认本次修改文件没有新增 lint 错误。
