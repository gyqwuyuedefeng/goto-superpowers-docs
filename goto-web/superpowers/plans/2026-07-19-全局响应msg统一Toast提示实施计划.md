# 全局响应 msg 统一 Toast 提示 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让 `goto-web` 对统一 Axios 响应中的非空 `msg` 自动显示且只显示一次 Toast，并按 `code=200/0` 区分成功和错误。

**Architecture:** 扩展 `vueToastificationNotice.js`，增加接收 `Vue.$toast` 与响应数据的 `notifyResponse` 入口，并复用现有 Toast 配置；`request.js` 在下载处理后调用该入口且仍原样返回 `response.data`。随后按后端返回契约移除已解析响应分支中的重复页面提示，保留普通 `success()`、前端校验、WebSocket 与 Promise rejection 提示。

**Tech Stack:** Vue 2.6、Axios 0.21、vue-toastification 1.7、Jest 23、ESLint 5、GitNexus CLI

## Global Constraints

- 仅当 `msg` 是非空字符串时提示；展示前执行 `trim()`。
- 仅严格数值 `code === 200` 显示 success Toast，严格数值 `code === 0` 显示 error Toast。
- `success()`、`successData(...)`、空白 `msg`、`code=2000`、未知响应码与非对象响应不自动提示。
- 不修改后端 `goto`、公共库 `freedom-public` 或接口返回文案。
- 不改变 Axios Promise 返回值、业务失败控制流、下载处理、401/403、超时或 HTTP 异常处理。
- 保留表单校验、前端条件提示、WebSocket 提示和 Promise rejection 的上下文提示。
- 不新增请求级关闭开关，不新增依赖。
- `goto-web/src/views/login.vue` 是用户现有未提交修改，本计划不得编辑、暂存或提交该文件。
- 修改现有 symbol 前必须运行 GitNexus upstream impact；已知 `notify` 与 `ifDownload` 风险均为 MEDIUM，没有 HIGH/CRITICAL，但执行时仍需复核并向用户报告。
- 每次提交前先尝试 GitNexus 变更检测；当前 CLI 若不提供 `detect-changes`，必须记录工具限制，并用 `git diff --check`、`git diff --name-only` 和定向测试核对范围。
- Git 提交信息使用中文，每个任务只提交任务列出的文件。

Modify `src/utils/vueToastificationNotice.js` 为以下结构；删除未使用的 `CommonUtils` import：

```js
const TOAST_OPTIONS = {
  position: 'top-right',
  timeout: 3000,
  closeOnClick: true,
  pauseOnFocusLoss: true,
  pauseOnHover: true,
  draggable: true,
  draggablePercent: 0.6,
  showCloseButtonOnHover: false,
  hideProgressBar: false,
  closeButton: 'button',
  icon: true,
  rtl: false
}

export default {
  notifySuccess(context, title, message) {
    this.notify(context, title, message, 'success')
  },
  notifyError(context, title, message) {
    this.notify(context, title, message, 'error')
  },
  notifyWarning(context, title, message) {
    this.notify(context, title, message, 'warning')
  },
  notifyInfo(context, title, message) {
    this.notify(context, title, message, 'info')
  },
  notifySimple(context, title, message, messageType) {
    if (!['success', 'error', 'warning', 'info'].includes(messageType)) return
    this.notify(context, title, message, messageType)
  },
  notifyResponse(toast, responseData) {
    if (
      responseData === null ||
      typeof responseData !== 'object' ||
      Array.isArray(responseData)
    ) return

    const message = typeof responseData.msg === 'string'
      ? responseData.msg.trim()
      : ''
    if (!message) return

    if (responseData.code === 200) {
      this.notifyWithToast(toast, message, undefined, 'success')
    } else if (responseData.code === 0) {
      this.notifyWithToast(toast, message, undefined, 'error')
    }
  },
  notify(context, title, message, messageType) {
    this.notifyWithToast(context.$toast, title, message, messageType)
  },
  notifyWithToast(toast, title, message, messageType) {
    let content = ''
    if (title) content += title
    if (message) content += '：' + message
    if (!content || !['success', 'error', 'warning', 'info'].includes(messageType)) return

    toast[messageType](content, { ...TOAST_OPTIONS })
  }
}
```
